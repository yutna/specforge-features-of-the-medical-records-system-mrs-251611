# User Stories with Acceptance Criteria

## US-01 — Patient registration and profile management

- **Derived from:** FR-01
- **Story:** As a user, I want patient registration and profile management so that deliver the intended business outcome.

### Acceptance criteria
- AC-01 — The system shall allow a patient to be found using HN, national ID, name, or phone number, and shall support duplicate patient detection during patient registration and profile management.
- BR-01 — Medication handling shall include: - Drug interaction checking - Allergy checking

### Supporting context
- **Workflow reference:** WF-01 — OPD encounter documentation workflow
- **Integration touchpoint:** INT-01 — Laboratory results integrated with medical records
- **Integration touchpoint:** INT-02 — PACS and image support
- **Business outcome:** deliver the intended business outcome

### Delivery notes
- Implement the story so that downstream teams can trace it back to FR-01.
- Preserve validation, error handling, and operational visibility for this story.
